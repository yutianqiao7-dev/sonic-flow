# SONIC FLOW - アクション音ゲー

ビートに乗って走る横スクロールのリズムランナー。タイミングよくジャンプすると、その動きに連動して音がどんどん気持ちよくなる。

## コンセプト: 「決めるほど音楽が育つ」

- 効果音を再生するのではなく、**すべての音を Web Audio API でリアルタイム合成**
- ヒット音はステージ固有のペンタトニック階段をコンボに合わせて駆け上がる → 決めるほど高揚する
- BGM はコンボ数に連動して変化:
  - コンボ 4+ : ハイハットが入る
  - コンボ 12+ : クラップ・ベースのグルーヴが増える
  - コンボ 24+ : ハイハット16分化・パッドのフィルターが全開に
  - ミス → 音がこもる (ローパスフィルターが閉じる)
- ヒット音は常にその曲と協和するので、どのタイミングで決めても気持ちいい

## ステージ (3種) — ジャンルがガラッと変わる

タイトルで選択。ジャンルごとにドラムパターン・ベースの奏法・和音の鳴らし方・
スウィング・サイドチェイン・音色・BPM・難易度がすべて変わり、別物の音楽になる:

| ステージ | ジャンル | BPM | 特徴 | 難易度 |
|---|---|---|---|---|
| MIDNIGHT | JAZZ | 100 | スイング・ウォーキングベース・ブラシドラム・Rhodes風コンピング | やさしい |
| NEON | EDM | 126 | 四つ打ち・強いサイドチェイン・オフビートのプラックベース・クラップ | ふつう |
| STORM | DnB | 150 | 高速ブレイクビート・リース/サブベース・ダークなスタブ | むずかしい |

## 遊び方

- キャラは自動で走る。**光るパッドが足元に来た瞬間に操作**する
- 操作は3種類:
  - **タップ = ジャンプ**。空中でもう一度タップ = **2段ジャンプ**(高いコインに届く)
  - **下スワイプ = スライディング**
- 障害物3種:
  - **トゲ (ピンク)** … ジャンプで跳び越える
  - **頭上バー (オレンジ)** … スライディングでくぐる
  - **敵 (紫)** … 1拍前にジャンプして着地で踏む(踏むと反動でもう一度跳ねる)
- **コイン (黄)** … ジャンプの弧・地面・2段ジャンプ専用の高所に配置。集めるとスコア+
- ジャンプ/スライドの持続はちょうど1拍なので、正しい操作は常に音楽のビートに乗る
- 判定: PERFECT (±65ms) / GOOD (±140ms) / 引っかかると MISS (つまずくが死なない。音がこもるのが罰)
- 1プレイ約60〜70秒。コースは毎回手続き生成され、後半ほど密度が上がる

## 開発

```bash
npm install
npm run dev      # 開発サーバー (スマホ実機で試すなら --host を付けて LAN の IP へアクセス)
npm run build    # 本番ビルド → dist/
```

## 構成

```
src/
  main.ts           # 画面遷移・ゲームループ・各モジュールの接続
  audio/
    engine.ts       # AudioContext・マスターチェーン・リバーブ
    music.ts        # BGM シーケンサー (ステージ別コード進行・ドラム・intensity 連動)
    sfx.ts          # ヒット音 (ペンタトニック階段) / コイン / 踏み / ミス音
  game/
    types.ts        # 共通型 (Entity, Coin) ・定数 (判定窓など)
    stages.ts       # ステージ定義 (BPM・キー・音色・色・難易度)
    chart.ts        # コース (障害物+コイン) の手続き生成
    input.ts        # タップ / スワイプ検出 (Pointer Events)
    game.ts         # ランナー物理 (2段ジャンプ/スライド)・踏み・判定・スコア
    renderer.ts     # Canvas 描画 (横スクロール・キャラ・障害物・コイン・HUD)
```

## ストア配布に向けて (次のステップ)

Capacitor でラップすると Android / iOS アプリとしてストアに出せる:

```bash
npm install @capacitor/core @capacitor/cli
npx cap init "SONIC FLOW" com.example.sonicflow --web-dir dist
npm run build
npx cap add android   # (iOS ビルドには Mac + Xcode が必要)
npx cap open android
```

技術メモ:
- タイミングはすべて `AudioContext.currentTime` 基準 (描画とは独立しており音ズレしない)
- アプリがバックグラウンドに行くと AudioContext ごと suspend され、完全に一時停止する
- ハプティクス: `navigator.vibrate` (Android のみ。iOS 対応は Capacitor Haptics プラグインで)
